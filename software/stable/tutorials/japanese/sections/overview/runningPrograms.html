プログラムの起動方法

sdlBasicのプログラムは単なるテキストファイルです。 プレインテキスト（.TXT)のタイプで保存できるエディターであればどんなエディターでも使えます。 sdlBasicで書かれたプログラムを起動には、sdlBasicの実行可能ファイル sdlbasic.exe(リーナっクスの場合sdlBasic）がカレントディレクトリーまたはパスのどこかにあればよい。 インクルードファイルが必要な場合はソースと同じディレクトリーになければいけません。

コマンドラインからのsdlBasic起動方法
ウインドウズまたはリーナックスのシェルから単に：

sdlbasic filename

とタイプします。 filenameのところにはあなたが走らせたいファイルの名前が入ります。
例えばmyprog.sdlbasという名のプログラムを走らせるのには、

sdlbasic myprog.sdlbas

とタイプしてください。

プログラムの起動の際に、インタープリターにいくつかのスイッチを渡すことができます。

--nosound : SDLのサウンドシステムを止める
--nosocket : SDLのソケットを止める
--debug : デバッグモードにする。
--nodefaults : デフォルトのディスプレイの起動を止める
--version : バージョンナンバーをプリントする
--help : sdlBrt のパラメータリストをプリントする
--copyright : コピーライトをプリントする
--license : ライセンスをプリントする

コマンドラインは次のようなものになるでしょう：

sdlbasic myprog.sdlbas --nosound

いくつかのリーナックスマシンで、サウンドデバイスにトラブルがある場合があります。 他のアプリケーションで問題のない場合、解決するには次の方法を試してみてください。

artsdsp -m sdlbasic myprog.sdlbas

ウインドウズまたはKDEからの プログラム起動

一番簡単な方法はイクステンションを関連付けることです。 （つまり、.sdlbasをsdlBrt.exeとを関連付ける。） そうすればダブルクリックすることによりsdlBasicのインタープリターを 自動的に起動することができます。
